# PMForge 1.1.0-rc.1 — "V1 Expansion" (Release Candidate)

**Git tag:** `v1.1.0-rc.1`  ·  **GitHub release title:** `PMForge v1.1.0-rc.1`

> **Release candidate.** This build is for evaluation and testing — not for
> data you can't afford to lose. Back up anything important and please report
> what you find. The packages are **unsigned**, so Windows SmartScreen and
> macOS Gatekeeper will warn on first launch (see Install).

PMForge is a **local-first** desktop application for project controls across
engineering, IT, construction, and administrative work. Your projects live in
encrypted files on your own machine — no account, cloud, or network required.

This is the **first release with click-installable native packages** for
Windows, macOS (Apple Silicon), and Linux, covering the full feature set below.

## Highlights

- **Project controls** — lifecycle status, budgets, stakeholders, timeline
  events, project settings, audit records, repair, backup, and local project
  files.
- **Scheduling** — CPM schedules with typed dependencies, lag, constraints,
  baselines, progress, Earned Value Management, named resource calendars,
  calendar-aware leveling, and Gantt charts; MSPDI import/export plus CSV,
  HTML, and report exports.
- **Charts** — 21 chart kinds across DAG, flow, matrix, and statistical
  engines, with in-app editing and vector PDF rendering.
- **Documents** — 25 project document kinds with schema-driven editing,
  bespoke PDF renderers, DOCX/ODT export, and combined reports with embedded
  chart visualisations.
- **Methodology packs** — Agile / Software-Dev views (Kanban, Backlog,
  Sprints, DORA metrics) and Process Excellence views (Six Sigma / DMAIC).
- **Embedded analytics** — production installers include the in-memory DuckDB
  analytics engine for portfolio rollups and local CSV/TSV, Parquet, and JSON
  data import. Portfolio cost totals are staged as integer minor units, then
  converted to display values after aggregation.
- **Security & compliance** — local Argon2id accounts, one-time recovery
  codes, SQLCipher-encrypted per-user `.pmforge` project databases, PDF/A
  validation, PAdES signing support, and the first tamper-evident audit-event
  hash-chain foundation. Project, chart, document, baseline, scenario,
  scenario-chart copy, document approval, scenario-promotion approval, and
  document signature actions now write `audit_events`, and compliance mode can
  verify the chain before opening a project. Project Settings can also export a
  private JSON audit verification report and raw repair-evidence artifact for
  review before manual audit-chain repair.
- **Scenario analysis foundation** — `.pmforge` databases now include scenario
  metadata storage for future what-if branches, including source-baseline
  linkage, a one-active-scenario-per-project constraint, and Wails bridge
  methods plus a Project Settings panel for create/list/edit/delete. Project
  Settings can now copy a live chart, optionally with a saved schedule
  baseline, into an isolated scenario partition, compare copied scenario data
  against its captured baseline, and promote copied scenario chart data back to
  a named schedule baseline. Copied scenario charts can also be opened in a
  dedicated scenario editor, compared, promoted, and saved without mutating the
  live source chart.

## Install

| Platform | File | How to install |
|---|---|---|
| Windows 10/11 (x86-64) | `PMForge-1.1.0-rc.1-amd64-setup.exe` | Run the installer → **More info → Run anyway** if SmartScreen appears |
| macOS (Apple Silicon) | `PMForge-1.1.0-rc.1-arm64.dmg` | Drag to Applications → first launch: right-click → **Open** |
| Debian / Ubuntu (x86-64) | `pmforge-1.1.0-rc.1-amd64.deb` | `sudo apt install ./pmforge-1.1.0-rc.1-amd64.deb` |
| Fedora / RHEL (x86-64) | `pmforge-1.1.0-rc.1-x86_64.rpm` | `sudo dnf install ./pmforge-1.1.0-rc.1-x86_64.rpm` |

Full per-platform steps: [docs/INSTALL.md](../INSTALL.md).

## Known limitations in this candidate

- **Unsigned builds.** The "unidentified developer" warnings are expected;
  code-signing/notarization is planned. The packages are safe to install.
- **Linux runtime libraries.** The `.deb` is built against Ubuntu 24.04+
  WebKit2GTK 4.1 with Wails' `webkit2_41` tag. Wails v2 still links GTK3;
  GTK4/WebKitGTK 6.0 support is a future framework migration. The `.rpm` is
  provided for testing — runtime on Fedora has not yet been validated.
- **Untagged developer builds omit DuckDB.** Use `make build` for release-like
  testing; direct untagged `go build` runs the analytics-unavailable fallback.

## Your data stays local

Each project is a SQLCipher-encrypted database under your user data folder.
Uninstalling does not delete your projects. Whole-disk encryption (FileVault /
BitLocker / LUKS) is still recommended as device-level protection.
